# Memory - February 25, 2026

## System Architecture

### Data Layer
- Canonical data directory: `~/.openclaw/workspace/data/options/trades.json`
- Schema: `{ version: 1, events: [...] }`
- Each event has: id, timestamp, account, ticker, option_type, event_type, contracts, price, strike, expiration, source_hash

### P/L Engine (plEngine.ts)
- Pure functions, computed at query time
- `computeRealizedPositions()` - groups by account+ticker+strike+exp, tracks contracts, calculates P/L when position closes
- `computePerformance()` - weekly/monthly/yearly aggregation
- `computeTickerPerformance()` - by ticker aggregation
- WIN if realized_pl > 0, LOSS otherwise
- Guard throws error if negative contract state detected

### API Endpoints
- `/api/trades` - CRUD
- `/api/trades/open` - open positions
- `/api/trades/realized` - closed positions with P/L
- `/api/trades/performance?period=` - aggregation by period
- `/api/trades/performance/ticker` - by ticker
- `/api/trades/import/schwab-csv` - CSV import preview
- `/api/trades/import/schwab-commit` - commit import

### Frontend
- Next.js app at `~/mission-control-next/`
- Trading tab with Performance panel
- CSV import for Schwab trades
- Event editing/deletion
